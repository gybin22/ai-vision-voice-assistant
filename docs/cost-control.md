# 成本控制策略

## 已采用策略

1. **只在用户提交问题时截取当前帧**  
   不上传连续视频流，避免高频视觉模型调用。

2. **前端图片压缩**  
   当前帧在前端通过 canvas 缩放到最长边 768px，并压缩为 JPEG。

3. **后端图片大小限制**  
   默认最大 800KB，超过直接拒绝。

4. **问题长度限制**  
   默认最大 500 字，避免异常长 prompt。

5. **输出 token 限制**  
   默认最大 500 tokens。

6. **单 session 限频**  
   默认 10 秒 1 次。

7. **单 session 日配额**  
   默认每天 30 次。

8. **单 IP 日配额**  
   默认每天 100 次。

9. **短上下文**  
   只保留最近 3 轮对话。

10. **请求缓存**  
   使用问题 hash + 图片 hash 作为 cache key，默认缓存 10 分钟。

11. **浏览器 ASR/TTS**  
   优先使用浏览器能力，避免额外云端语音识别和 TTS 成本。

12. **Mock 模型**  
   本地开发默认不调用真实模型。

## 未采用但可后续扩展

- 端侧小模型做图像预筛选；
- 向量库做图片语义缓存；
- 用户登录和付费配额；
- 动态路由到不同价格模型；
- VAD 语音活动检测；
- WebRTC 实时语音视频流。
