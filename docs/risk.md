# 核心风险

## 1. 浏览器权限风险

用户拒绝摄像头或麦克风权限后，应用无法完成视觉和语音输入。应在前端提供明确提示，并保留文字输入。

## 2. 语音识别兼容性风险

浏览器 Web Speech API 在不同浏览器中支持程度不同。MVP 使用文字输入兜底。

## 3. 移动端兼容性风险

移动端尤其是 iOS Safari 对 autoplay、playsinline 和权限触发时机敏感。摄像头启动应由用户点击触发。

## 4. 模型幻觉风险

多模态模型可能描述画面中不存在的内容。Prompt 中要求“只描述能确定看到的内容，不确定则说明不确定”。

## 5. 成本失控风险

恶意高频请求、大图上传、长输出都可能造成成本失控。通过限频、配额、图片限制、缓存和 token 限制缓解。

## 6. 隐私风险

摄像头图片可能包含敏感信息。默认不保存图片，不在日志中输出图片内容。

## 7. API Key 泄露风险

模型 API Key 只能放在后端环境变量，前端不得直接调用模型接口。
