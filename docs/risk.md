# 风险与后续优化

## 1. 浏览器权限风险

### 风险

摄像头和麦克风权限必须由用户授权。用户拒绝权限后，实时语音和视觉理解无法使用。

### 当前处理

- 保留手动输入作为兜底；
- 摄像头作为视觉上下文自动准备，而不是让用户理解底层开关；
- 页面提示浏览器是否支持语音识别。

### 后续优化

- 增加权限引导页；
- 对 Safari、移动端浏览器提供专门提示；
- 支持“纯文字模式”。

---

## 2. Web Speech API 兼容性风险

### 风险

浏览器 Web Speech API 支持不完全，尤其 Firefox、部分 Safari 和移动端环境可能不可用或不稳定。

### 当前处理

- 实时语音不可用时提示用户切换手动输入；
- 保留文字输入和发送按钮；
- AI 播报使用浏览器 SpeechSynthesis。

### 后续优化

- 接入云端 ASR；
- 增加 VAD 语音活动检测；
- 增加设备检测和兼容性提示。

---

## 3. 语音自动提交误触发风险

### 风险

实时模式下，语音识别可能因为背景音、停顿、半句话而自动提交，导致体验不佳和成本增加。

### 当前处理

- 用户停顿约 1 秒后才提交；
- AI thinking/speaking 期间暂停或限制提交；
- 保留手动模式；
- 后端限制请求频率和配额。

### 后续优化

- 加入更准确的 VAD；
- 增加“确认后发送”可选模式；
- 支持用户自定义停顿时间。

---

## 4. 视觉理解误判风险

### 风险

多模态模型可能误识别物体、动作或状态，也可能产生幻觉。

### 当前处理

Prompt 中要求：

- 不确定时说明“不太确定”或“看起来像”；
- 只回答用户问题相关内容；
- 不逐帧描述；
- 不强行猜测用户情绪、身份、职业或意图；
- 不把视觉上下文当成每次必须使用的信息。

### 后续优化

- 引入目标检测或端侧预分析；
- 对高风险场景增加安全提示；
- 根据用户反馈优化 prompt。

---

## 5. 成本失控风险

### 风险

高频请求、大图、多帧、长文本、长回答都可能造成模型成本增加。

### 当前处理

- 前端低频抽帧；
- 最多 15 帧；
- 前端图片压缩；
- 后端限制单图大小、总图大小、问题长度、输出 tokens；
- 后端限流和每日配额；
- 短上下文；
- 缓存；
- 用户 Tokens 余额校验；
- 失败不扣费；
- 本地开发默认 Mock 模型。

### 后续优化

- 增加更细粒度的用户套餐；
- 加入模型动态路由；
- 加入管理后台查看成本报表；
- 加入异常成本预警。

---

## 6. 模型 API 认证与上游稳定性风险

### 风险

真实模型调用可能遇到：

- API Key 错误；
- profile 未启用；
- base-url 错误；
- 模型无权限；
- 上游网关偶发 401/429/5xx；
- 网络超时。

### 当前处理

- 模型 Key 只放后端环境变量；
- Mock 和真实模型分离；
- 失败请求记录日志且不扣费。

### 后续优化

- 对空 body 的偶发 401/5xx 做一次短重试；
- 增加上游 requestId 记录；
- 增加备用模型；
- 增加服务健康检测。

---

## 7. 隐私风险

### 风险

摄像头画面可能包含人脸、房间环境、证件、屏幕内容等敏感信息。

### 当前处理

- 不保存用户上传的图片帧；
- 不在历史记录中保存图片；
- 不在日志中打印图片 Base64；
- 会话历史只保存文字和模型调用元数据；
- 提供清空历史功能。

### 后续优化

- 增加“暂停视觉上下文”开关；
- 增加本地马赛克或隐私遮挡；
- 增加用户隐私提示和确认协议。

---

## 8. 充值与计费风险

### 风险

当前充值为 demo 模拟充值，不能直接用于生产。真实商业化还需要支付回调、订单状态、验签、反作弊和退款处理。

### 当前处理

- 模拟充值只作为 MVP 展示；
- 数据库已拆分账户表、流水表、请求日志表；
- AI 请求扣费与真实 usage 绑定；
- 失败请求不扣费。

### 后续优化

- 接入微信支付、支付宝或 Stripe；
- 增加订单表和支付回调验签；
- 增加管理员对账功能；
- 增加退款和补偿机制。

---

## 9. 数据一致性风险

### 风险

Tokens 扣费涉及账户余额、消费流水和 AI 请求日志，必须保证一致性。

### 当前处理

- 后端使用事务；
- 扣费时锁定用户 token account；
- 成功请求写入交易流水和 request log；
- 失败请求记录 FAILED 日志，不扣费。

### 后续优化

- 增加幂等键；
- 增加定时对账任务；
- 增加后台修复工具。

---

## 10. 部署风险

### 风险

浏览器摄像头和麦克风在非 HTTPS 环境下通常受限。生产部署还需要 CORS、密钥、数据库和日志配置。

### 当前处理

- 本地通过 `localhost` 开发；
- 后端 CORS 支持白名单配置；
- 所有敏感配置通过环境变量读取。

### 后续优化

- 使用 HTTPS；
- 独立部署前后端；
- 增加 Nginx 反向代理；
- 增加日志脱敏和监控告警。
